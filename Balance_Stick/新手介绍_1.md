# Balance_Stick 小白介绍文档

这份文档给第一次接触平衡控制和 ESP32 的同学准备，尽量用直白的话讲清楚：

- 这个项目是做什么的
- 用到了哪些核心算法
- 你可以通过它实现什么功能
- 怎么快速跑起来、怎么安全调参

---

## 1. 这个项目是做什么的？

`Balance_Stick` 是一个基于 **ESP32-C3 + MPU6050 + 电机** 的平衡控制项目。

它的核心思路是：

1. 读取 IMU（姿态传感器）数据
2. 估算当前倾角
3. 用 PID 算法算出电机该给多大力度（PWM）
4. 实时驱动电机纠正倾斜

主循环在 `src/main.cpp`，执行链路如下：

`IMU采样 -> 控制计算 -> 死区补偿 -> 电机输出 -> Web状态上报`

---

## 2. 项目有哪些功能？

### 2.1 实时平衡控制

- 实时读取 MPU6050 数据（角度、角速度）
- 通过滤波减少噪声和抖动
- 通过 PID 计算电机输出
- 支持倾倒保护和手势开关，避免危险输出

### 2.2 网页调参（不用反复改代码）

设备会开启热点：

- SSID：`ESP32C3_Balance`
- 密码：空（默认无密码）

连上后打开网页，可做这些事：

- 实时修改 `kp/ki/kd`
- 修改目标平衡角 `keepAngle`
- 修改死区补偿 `deadzone`
- 一键保存参数到芯片（掉电不丢）
- 一键恢复默认参数
- 用当前姿态做“校准平衡点”

另外还有两个辅助页面：

- `/motor`：查看/切换电机方向（反转）
- `/deadzone`：手动输出 PWM，专门调试死区

### 2.3 参数持久化

项目用 ESP32 的 `Preferences`（NVS）保存关键参数：

- `kp/ki/kd`
- `keepAngle`
- `deadzone`
- 电机方向反转状态

重启后会自动加载。

## 3. 用到了哪些算法？（小白版）

> 这一部分不讲复杂推导，只讲“它解决什么问题”。

### 3.1 低通滤波（去抖动）

传感器原始数据会抖动，直接拿来控制会让电机发疯。
所以先做一阶低通（IIR）平滑：

- 加速度角度做低通
- 陀螺角速度做低通
- 对突变数据做简单异常值过滤（超阈值就不采用）

这样能明显减少噪声。

### 3.2 互补滤波（项目里保留）

互补滤波会把：

- 陀螺仪“短期变化快”的优势
- 加速度计“长期不漂移”的优势

合在一起。项目里还做了**动态 alpha**，运动快慢不同时权重会变。

### 3.3 卡尔曼滤波（当前主用）

卡尔曼滤波本质是“更聪明的融合器”，会估计：

- 当前角度
- 陀螺仪偏差（bias）

在本项目中，PID 使用 `kalmanFilter()` 的角度输出作为主要控制输入。

### 3.4 PID 控制（核心控制器）

项目的控制输出是：

`motor_pwm = kp * bias + ki * integrate + kd * gyroX_lp`

其中：

- `bias`：当前角度与目标角（`keepAngle`）的差
- `integrate`：累计误差（项目里做了积分限幅，防止积分爆炸）
- `gyroX_lp`：低通后的角速度（相当于 D 项）

### 3.5 死区补偿

电机在很小 PWM 下可能“叫不动”。
所以项目会把过小的非零输出抬到 `deadzone` 附近，解决“算出来了但电机不转”的问题。

## 4. 安全与保护机制

为了避免误动作，项目做了几层保护：

1. **上电先熄火**：初始化后先 `flameoutMotor()`
2. **倾倒保护**：角度超范围时直接停转并清零积分
3. **手势开关**：通过陀螺 Y 轴阈值开启/关闭控制
4. **手动模式隔离**：`/deadzone` 手动调试与自动控制分开

如果你在调参时看到电机突然停转，多半是保护触发了，这通常是正常现象。

## 5. 快速上手（从 0 到能动）

### 第一步：准备环境

- VS Code + PlatformIO IDE
- 硬件：ESP32-C3、MPU6050、电机驱动与电机、电源

### 第二步：编译

在项目根目录执行：

`pio run -e airm2m_core_esp32c3`

### 第三步：烧录

`pio run -e airm2m_core_esp32c3 -t upload`

### 第四步：连接热点

设备启动后，手机/电脑连接 `ESP32C3_Balance`。

### 第五步：打开网页调参

在浏览器访问设备 IP（常见为 `192.168.4.1`），进入主页面调整参数。

## 6. 小白调参建议（非常实用）

建议按这个顺序调，不容易炸：

1. `ki = 0`，先只调 `kp`
2. `kp` 从小到大加，直到能明显纠偏但不剧烈震荡
3. 再加 `kd` 抑制抖动
4. 最后少量加 `ki` 消除慢性偏差
5. 调 `deadzone`，让电机小输出也能起转

经验上，调参宁可慢一点，也不要一步加太大。

## 7. 常见问题

### Q1：电机方向反了怎么办？

去 `/motor` 页面切换反转，确认后保存。

### Q2：网页改了参数，重启又没了？

你可能只点了“更新”没点“保存”。
只有保存才会写入 NVS。

### Q3：为什么一开始电机不转？

可能原因：

- 保护开关未满足（倾倒/手势）
- PWM 还在死区内
- 电机接线或电源问题

### Q4：控制很抖怎么办？

先降低 `kp`，适当提高 `kd`，并检查 IMU 固定是否牢靠。
